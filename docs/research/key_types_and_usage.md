# Key Types and Usage Research

This document explains how the RustDesk protocol handles keys for server communication and peer authentication.

## 1. The Server Key (Root of Trust)

The primary security parameter in a RustDesk deployment is the **Server Ed25519 Public Key** (found in `id_ed25519.pub` on the server).

- **Role:** This key is used by the client to verify all signatures provided by the rendezvous server (`hbbs`).
- **Reuse:** The official client uses this same key for two distinct verification steps:
    1.  **TCP KeyExchange:** Verifying the signature of the server's ephemeral key on port 21116.
    2.  **Peer Handshake:** Verifying the `SignedId` message, which contains the peer's public key signed by the server.

## 2. Key Acquisition Flow

The official client (`src/common.rs:get_key`) retrieves the key in the following priority:
1.  **License Key:** If a Pro license is present, it may contain an embedded key.
2.  **User Configuration:** The "Key" field in the client's Network settings.
3.  **Hardcoded Default:** If nothing else is provided, it uses:
    `OeVuKk5nlHiXp+APNn0Y3pC1Iwpwn44JGqrQCsWqmBw=`

## 3. Verification Process

### TCP KeyExchange (hbbs)
When connecting via TCP to `hbbs`, the server sends a `KeyExchange` message. The client performs:
```rust
// common.rs:secure_tcp
let their_pk_b = sign::verify(&ex.keys[0], &rs_pk)?;
```
Here, `rs_pk` is the Server Ed25519 Public Key. This ensures the client is talking to the legitimate rendezvous server.

### Peer Authentication (SignedId)
During the encrypted handshake with a peer, the client receives a `SignedId`. It performs:
```rust
// client.rs:secure_connection
if let Ok((id, pk)) = decode_id_pk(&signed_id_pk, &rs_pk) { ... }
```
Again, `rs_pk` is the Server Ed25519 Public Key. The server acts as a Certificate Authority (CA), signing the peer's public key so the client can trust it without a direct key exchange with the peer.

## 4. Summary Table

| Key Type | Source | Purpose in Protocol |
| :--- | :--- | :--- |
| **Server Public Key** | User / `id_ed25519.pub` | Verifies hbbs and verifies peers via `SignedId`. |
| **Server Ephemeral Key** | Generated by hbbs | Used for `ChaCha20-Poly1305` transport encryption. |
| **Peer Public Key** | Inside `SignedId` | Used for E2E encryption between clients. |

## Conclusion for rustdesk-cli

The `--key` parameter passed to `rustdesk-cli` MUST be the **Server's Ed25519 Public Key**. 
- If the user provides `SWc0NIWF0wR7kd8rHdGNaCHXtp7dirUImEtrVmRfQdc=`, this is the key used for ALL signature checks.
- If verification fails at the TCP `KeyExchange` step, it is likely that the provided key does not match the `id_ed25519` key pair being used by the server at `115.238.185.55`.
